## Approach — AEO-First Across 4 Channels

Every piece of content (product page, blog, social post, ad) is written so that:
1. A human reads it and buys
2. An AI engine extracts it and cites Secuview as the source

### Pillar 1: Product Page Optimization (AEO + Schema)
- Rewrite product descriptions as extractable answer blocks (40-60 word passages)
- Add definition block: "What is a solar 4G security camera?"
- Add comparison section: Single vs Dual Lens — table format for AI extraction
- Add FAQ section: 5-7 real questions buyers ask
- Add `llms.txt` at site root for AI crawlers
- Ensure Product schema has: price, availability, SKU, brand, image, GTIN/MPN
- Verify robots.txt allows GPTBot, PerplexityBot, ClaudeBot, Google-Extended

### Pillar 2: Hero Blog Post (AEO Magnet)
- Title: "Solar 4G Security Camera Qatar 2026: Complete Guide for Villas, Construction Sites & Remote Properties"
- Structure: definition block → comparison table → use-case blocks → FAQ → buyer's guide
- Each section written as self-contained answer (extractable by AI)
- Statistics + expert framing for +40% AI citation boost
- Internal links to all 3 product pages

## AEO Requirements (Non-Negotiable)

Every piece of content must pass these checks:
- [ ] Lead with direct answer (don't bury the point)
- [ ] Key passages 40-60 words (optimal for AI extraction)
- [ ] One clear idea per paragraph
- [ ] Statistics with sources (+40% citation boost)
- [ ] FAQ section with natural-language questions
- [ ] Tables for comparison content (AI loves structured data)
- [ ] Expert framing (named author, credentials)
- [ ] Freshness signal (date + "last updated")
- [ ] Machines can read it (no JS walls, no gating)
- [ ] Robots.txt allows GPTBot, PerplexityBot, ClaudeBot, Google-Extended
